# Composition patterns & originality rules

## Base structure types

Pick one structure; don't blend too many.

### Workflow

For: input → process → output, content production, AI workflows, automation chains.

How: input on the left, yoyo or a strange machine processing in the middle, output on the right, an orange arrow for the main flow.

### System slice

For: information sources, filters, databases, renderers, a slice of an agent system.

How: draw only 3–5 core modules; yoyo performs one key action among them.

### Before / after

For: chaos vs order, manual vs automated, scattered vs gathered, anxious vs calm.

How: messy on the left, calm on the right, an orange arrow between. yoyo can be more exaggerated here.

### Character state

For: user pain points, creator states, too-many-tools, information anxiety, stuck → running.

How: 2–4 small states, one short label each.

### Concept metaphor

For: content factory, information warehouse, mental black box, workflow machine, automated daily report.

How: one big strange object or machine, a little input, one output. Make it memorable.

### Method layers

For: a methodology framework, system layers, a capability stack, a layered content system.

How: stacked boxes, not a formal pyramid; yoyo building or carrying bricks beside them.

### Map / route

For: idea → launch, user path, content hand-off path, learning route.

How: one winding path, a few nodes, yoyo pulling the line or walking it.

### Mini comic

For: failure → success, the real process, a wry observation, before/after of using something.

How: 2–4 small panels, one action per panel.

## How to invent an original metaphor

Invent a fresh metaphor from the current article every time; never reuse an old one.

### Three steps

1. Turn the abstract concept into a **physical action**: stuck, leaking, getting heavier, sorting, settling, fermenting, opening a door, folding, unpacking, flowing back.
2. Turn the system structure into a **low-tech object**: a broken machine, cardboard box, drawer, water pipe, mailbox, weird dial, scale, well, ladder, odd workstation.
3. Make **yoyo carry the action** with its tentacles: not standing beside it, but stuck in the machine, pulling the wrong line, guarding a gate, hauling, patching, weighing, holding the ladder, recording, plugging something into a strange device.

### Object pool

- cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, pipe, ball of string, gate, dial, black box, hole-punch, pasta press, clothesline, strange workstation.
- Use only 1–2 at a time; don't pile them up.

### yoyo action pool

- pull, haul, stuff, scoop, press, weigh, stitch, cut, twist, guard, push, catch, unpack, mark, recycle — done with tentacles.
- The action must serve the core idea; don't be weird just to be weird.

## Anti-cliché rule

`assets/yoyo-reference.png` exists only to keep yoyo on-model (proportions, color, face) — not as a composition to copy.

Reinvent the metaphor for every article, even for repeated themes. A "hand-off path" doesn't have to be a route — it could be yoyo hooking the tail of one piece of content onto a doorknob. "One input, many outputs" doesn't have to be a fish — it could be yoyo pressing one paper ball into several shapes. Avoid defaulting to generic conveyor-belt / funnel / box-sorting clichés unless the article truly calls for it.
