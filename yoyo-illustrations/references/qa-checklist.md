# QA Checklist

## Must pass

- It is 16:9 horizontal.
- The background is clean white.
- yoyo is present and on-model (soft purple octopus, dot eyes, small smile, eight tentacles).
- yoyo performs the core action — not just decoration.
- yoyo is the only purple/pink subject; nothing else carries those colors.
- It invents a fresh metaphor for this article, not a recycled cliché.
- The frame is quietly weird, creative, interesting.
- Clean and sparse; the subject is under about 60% of the canvas.
- One image explains only one core structure.
- Annotations are few, short, and readable.
- Annotations are in the same language as the source article.
- Orange is used only for the main path or arrows.
- Red is used only for key points, problems, reminders, or results.
- Blue is used only for secondary notes, feedback, or system state.

## Failure signals

Regenerate or locally edit if you see:

- A top-left title ("Common Pits / Workflow / System Architecture / Roadmap").
- yoyo looks like a mascot, sticker, or over-cute cartoon.
- It looks like a PPT, course slide, or formal flowchart.
- Too many elements, arrows, or nodes.
- Text has turned into long paragraphs of explanation.
- The background has paper texture, shadow, gradient, beige, or noise.
- A realistic UI screenshot or techy interface.
- Annotations have bad typos or are unreadable, or are in the wrong language.
- The frame is stiff, with no quietly-weird metaphor.
- Purple or pink has leaked onto props or background.

## Iteration moves

- Too plain: make yoyo the subject of the action; add one strange-but-valid metaphor.
- Too busy: drop nodes; keep one action and 3–5 short labels.
- Too cute: emphasize deadpan, calm expression, not-a-mascot, not-cute.
- Too PPT: remove title, borders, neat grid, and excess arrows; switch to a hand-drawn scene.
- Wrong language: regenerate the labels in the article's language.
- Color leak: keep purple/pink on yoyo only; recolor everything else black.
- Bad text: prefer a local edit; if there are many errors, regenerate with fewer labels.

## Delivery judgment

A good image should first read as "a little weird," then become clear within one second.

If the first impression is a tutorial page rather than a quietly-weird product sketch on blank paper, it fails.
